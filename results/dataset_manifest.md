# Dataset Manifest — General Navigation Model VLNVerse Training Data

## What was generated

A trajectory dataset was generated by replaying expert navigation episodes
from the Vision-Language Navigation Verse (VLNVerse) benchmark inside
NVIDIA Isaac Sim (version 5.1.0).

The robot does **not** explore randomly.
It follows a predefined expert trajectory for each episode.
At each step, NVIDIA Isaac Sim records the sensor output and robot state.

## Trajectory folder structure

Each trajectory is stored in one folder under `datasets/vlntube/train/`
or `datasets/vlntube/val/`.

Folder name format:
```
<scene>_<scene>_<episode_id>_<variant>
```
Example: `kujiale_0092_kujiale_0092_0_3`

Contents of each folder:

| File | Description |
|------|-------------|
| `0.jpg`, `1.jpg`, … | Red-Green-Blue (RGB) camera frames at 96×96 pixels, one per step |
| `traj_data.pkl` | NumPy arrays: `position` (N, 2) and `yaw` (N,) |
| `episode_info.json` | Episode metadata: `scan`, `episode_id`, `goal_pos`, `goal_radius`, `n_steps` |
| `instruction.txt` | Natural-language instruction (language annotation for Track B) |

---

## traj_data.pkl fields

The pickle file contains a Python `dict` with the following keys:

| Key | Type | Shape | Description |
|-----|------|-------|-------------|
| `position` | `numpy.ndarray float32` | `(N, 2)` | Floor-plane x, y in metres for each step |
| `yaw` | `numpy.ndarray float32` | `(N,)` | Robot heading angle in radians for each step |

Additional labels derived **at training time** by `GNMDataset` (not stored in the file):

| Derived label | How derived |
|---------------|-------------|
| Action / local waypoint target | `position[obs_idx + 1] − position[obs_idx]`, normalised |
| Distance-to-goal target | `(goal_idx − obs_idx) / max_goal_dist`, clamped to [0, 1] |
| Observation stack | Last `context_size` frames concatenated along channel axis |
| Goal image | Frame at `goal_idx` |

---

## episode_info.json fields

| Field | Type | Description |
|-------|------|-------------|
| `scan` | string | Scene identifier, e.g. `kujiale_0092` |
| `episode_id` | string | Unique episode string, e.g. `kujiale_0092_0_3` |
| `goal_pos` | list[float, float] | Goal position [x, y] in metres |
| `goal_radius` | float | Success threshold — robot must stop within this radius (3.0 m) |
| `n_steps` | int | Number of recorded frames in this trajectory |

## Label generation process

Labels are generated **automatically** from simulator data.
No manual annotation is required.

1. NVIDIA Isaac Sim opens the VLNVerse Universal Scene Description (USD) scene.
2. The robot follows the expert trajectory from the VLNVerse episode.
3. At each step, Isaac Sim records:
   - Red-Green-Blue camera frame (rendered at full resolution, then resized to 96×96 at training time)
   - Robot position (x, y) on the floor plane
   - Robot heading yaw angle
4. The generation script (`scripts/gnm/04_generate_data.py`) writes each step
   as a JPEG frame and saves `traj_data.pkl` and `episode_info.json`.

The action standard deviation used during training: `[0.115528, 0.115528]` (symmetric).

---

## Did the robot explore randomly?

**No.**
The robot replays predefined expert trajectories from the Vision-Language
Navigation Tube / Vision-Language Navigation Verse episodes.
The expert path is planned inside the VLNVerse navigation graph.
The robot follows this path exactly during Isaac Sim data generation.

## Scene-level generalisation split

The standard split above uses the prebuilt VLNVerse trajectory holdout.
A stricter scene-level holdout split is defined in
`configs/gnm/splits/scene_holdout_kujiale_0271.yaml`:

| Role | Scenes | Trajectories |
|------|--------|-------------|
| Train | kujiale_0092, kujiale_0203, kujiale_0118 | 191 |
| Scene holdout test | kujiale_0271 | 47 |

This tests whether the model generalises to an **entirely unseen floor-plan**.
The full training run on this split is pending.
The split config and data-count verification are complete.

## Reproduction

To regenerate the full dataset:

```bash
conda run -n isaac python scripts/gnm/04_generate_data.py
```

To inspect trajectory labels:

```bash
python3 scripts/gnm/inspect_trajectory_labels.py \
    --data-root datasets/vlntube \
    --split train \
    --limit 5
```

To verify the scene holdout split:

```bash
python3 scripts/gnm/check_scene_holdout_split.py \
    --data-root datasets/vlntube \
    --split-config configs/gnm/splits/scene_holdout_kujiale_0271.yaml
```
