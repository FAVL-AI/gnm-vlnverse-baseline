# Template Analysis: vlntube_fleetsafe Instructions

## Generation pipeline

Instructions in `datasets/vlntube/train/` and `datasets/vlntube/val/` were generated by the
VLNTube pipeline (`external/VLNTube/instube/`) using the **Gemini API** fed rendered trajectory
image sequences. They are stored in the upstream benchmark release `Eyz/VLNVerse_data` on
HuggingFace, then copied verbatim into `instruction.txt` by `scripts/gnm/vlntube_runner.py`.

Generation prompt (from `instube/prompt.py`) required:

1. **Landmarks** — areas, rooms, furniture the agent passes  
2. **Spatial information** — direction of landmarks relative to agent  
3. **Actions** — command verbs (walk, turn, proceed, stop)  
4. **End point** — explicit description of final stopping position  

## Instruction types observed

| Type | Count | Notes |
|------|-------|-------|
| spatial_route_instruction | 235 | Landmarks + spatial refs + explicit "Stop …" endpoint |
| visual_landmark_grounded | 18 | Landmarks + spatial refs + "stopping …" endpoint (regex variant) |
| generic_navigation_prompt | 0 | None present |
| coordinate_or_pose_derived | 0 | No x= y= patterns found |
| duplicated_across_episodes | 0 | All 253 SHA-256 values distinct |

## Structural pattern

All instructions follow the instube prompt template:
- Start: describe starting position or first visible landmarks
- Middle: route description with spatial cues ("on your left", "turn right")
- End: explicit goal statement ("Stop in front of …", "stop near …", "stopping just inside …")

The endpoint description was generated by Gemini after viewing the goal-area frames. This is
**standard VLN task formulation** (the instruction tells you what to navigate to) and is NOT
retrieval-system leakage. The goal position (`goal_pos`) was defined geometrically by the
scene-graph pipeline before any instruction was generated.

## No template substitution patterns

No variable-substitution markers (`{room}`, `{{target}}`, etc.) were found. Each instruction is
a unique natural-language paragraph. There are no exact duplicates across the 253 episodes.

## Independence assessment

- `goal_pos` origin: A* path planning on scene graph geometry — does NOT depend on instruction
- Instruction origin: Gemini API viewing rendered trajectory frames — does NOT depend on `goal_pos` coordinates
- Track B CLIP retrieval: implemented AFTER the VLNTube data was generated — no circular dependency
