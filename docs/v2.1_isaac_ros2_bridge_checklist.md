# v2.1 — Live Isaac ROS 2 Bridge Checklist

## Scope

v2.1 adds the first live Isaac ROS 2 verification layer on top of the v2.0 dry-run pipeline.

**What v2.1 does:**
- Checks Isaac Sim ROS 2 bridge availability.
- Verifies the five required live topics are published.
- Provides a Yahboom M3 Pro placeholder control scene checklist.
- All checks exit 0 without Isaac Sim running (CI-safe by default).

**What v2.1 does not do:**
- No live GNM inference or control.
- No FleetSafe CBF-QP shielding.
- No Yahboom hardware integration (placeholder only).

---

## Prerequisites

- Isaac Sim 4.x with the ROS 2 Bridge extension enabled.
- ROS 2 Humble or Jazzy installed on the host.
- Python 3.10+.

---

## Starting Isaac Sim with the ROS 2 Bridge

1. Launch Isaac Sim from its install directory:

   ```bash
   ~/.local/share/ov/pkg/isaac-sim-*/isaac-sim.sh
   ```

2. In the Isaac Sim menu bar, open **Window → Extensions**.

3. Search for `ROS2 Bridge` and toggle it enabled.  
   The bridge starts automatically and begins publishing on the configured `ROS_DOMAIN_ID`.

4. Verify the bridge is active from a separate terminal:

   ```bash
   source /opt/ros/humble/setup.bash
   ros2 topic list
   ```

   You should see at least `/rosout` and, once a robot stage is loaded, the
   sensor topics listed below.

5. If the robot stage is loaded, confirm the required topics are present:

   ```bash
   ros2 topic echo /camera/image_raw --once
   ros2 topic echo /odom --once
   ```

---

## Bridge Availability Check

```bash
bash scripts/gnm/check_isaac_bridge.sh
```

With strict mode (requires Isaac Sim to be running):

```bash
bash scripts/gnm/check_isaac_bridge.sh --strict
```

The script exits 0 in both cases when Isaac Sim is not running, unless `--strict` is
passed. In CI, run without flags.

---

## Required Live Topics

| Topic | Source in Isaac Sim |
|---|---|
| `/camera/image_raw` | Camera sensor prim with ROS 2 camera publisher |
| `/odom` | Differential drive articulation with odometry publisher |
| `/tf` | Isaac Sim TF tree publisher (ROS 2 bridge built-in) |
| `/scan` | Lidar sensor prim with ROS 2 laser scan publisher |
| `/cmd_vel` | Twist subscriber wired to the robot's drive joints |

### Verify topics

```bash
python3 scripts/gnm/verify_live_topics.py
```

With strict mode (all five topics must be present):

```bash
python3 scripts/gnm/verify_live_topics.py --strict
```

---

## Dry-run / CI Mode

Both `check_isaac_bridge.sh` and `verify_live_topics.py` exit 0 when ROS 2
is not installed or when Isaac Sim is not running. Use `--strict` only when
Isaac Sim is confirmed running and the robot stage is loaded.

```bash
bash scripts/gnm/check_isaac_bridge.sh
python3 scripts/gnm/verify_live_topics.py
```

Both commands complete in CI without any simulator running.
