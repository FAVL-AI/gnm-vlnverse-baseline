# v2.4 — First Valid Yahboom Isaac rosbag2 Episode

## Purpose

v2.4 records the first valid rosbag2 episode from the Yahboom ROSMASTER M3 Pro in
Isaac Sim.

A recording is only valid if all five canonical topics publish non-zero messages
for the full episode duration. An invalid recording cannot be converted to GNM
training data. The gate is enforced automatically by the recording script — it
will refuse to start if topic verification fails.

---

## The Dependency Chain

```
verify_yahboom_live_topics.py --strict → must pass
        ↓
collect_yahboom_episode.sh → records episode_001
        ↓
validate_yahboom_episode.py → confirms non-zero counts on all five topics
        ↓
Gate A6 open → v2.5 (rosbag2 → GNM dataset conversion) can begin
```

No step in this chain can be skipped. Each is a hard prerequisite for the next.

---

## What Is a Valid Episode?

| Requirement | Check |
|---|---|
| All five canonical topics present in bag | `validate_yahboom_episode.py` |
| All five topics have message_count > 0 | `validate_yahboom_episode.py` |
| No Nova Carter topic names in the bag | `validate_yahboom_episode.py` |
| Bag contains at least 30 seconds of data | `validate_yahboom_episode.py` |
| Episode metadata JSON written | `collect_yahboom_episode.sh` |
| Validation report JSON written | `validate_yahboom_episode.py` |

### Minimum message counts (guideline)

| Topic | Typical Hz | Min messages in 60 s |
|---|---|---|
| `/camera/image_raw` | 30 | 1800 |
| `/odom` | 50 | 3000 |
| `/tf` | 100 | 6000 |
| `/scan` | 10 | 600 |
| `/cmd_vel` | 10 | 600 |

These are guidelines. The validator enforces message_count > 0, not a minimum
rate. Minimum episode length is 30 seconds.

---

## Pre-conditions (Gate A5 must pass)

Before running the recording script, all of the following must be true:

1. Isaac Sim is open and the Yahboom M3 Pro USD stage is loaded.
2. The ROS 2 Bridge extension is enabled in Isaac Sim.
3. Isaac Sim is in Play mode (press the Play button in the toolbar).
4. All five OmniGraph action graphs are connected:
   - Camera publisher → `/camera/image_raw`
   - Odometry publisher → `/odom`
   - TF publisher → `/tf`
   - LiDAR publisher → `/scan`
   - Twist subscriber → `/cmd_vel`
5. In a terminal, the topic gate verifier passes:
   ```bash
   python3 scripts/gnm/verify_yahboom_live_topics.py --strict
   ```
   The script must print `[OK] All canonical topics present.` and exit 0.

If any of these conditions are not met, the recording script will detect the
failure and refuse to start.

## Recording Procedure

### Step 1 — Verify topics (mandatory gate)

```bash
python3 scripts/gnm/verify_yahboom_live_topics.py --strict
```

All five topics must pass. Do not proceed until this exits 0.

### Step 2 — Record episode_001

```bash
bash scripts/gnm/collect_yahboom_episode.sh \
  --episode-name episode_001 \
  --duration 60
```

The script will:
1. Re-run the topic gate (`verify_yahboom_live_topics.py --strict`) internally.
2. Refuse to record if the gate fails.
3. Call `ros2 bag record` on the five canonical topics for 60 seconds.
4. Write `datasets/gnm_fleetsafe_rosbags/episode_001/episode_metadata.json`.

During recording, drive the robot in Isaac Sim toward a goal location. The episode
should contain a navigable path — not just the robot standing still.

### Step 3 — Validate the episode

```bash
python3 scripts/gnm/validate_yahboom_episode.py \
  --episode-path datasets/gnm_fleetsafe_rosbags/episode_001
```

The validator checks:
- All five canonical topics are in the bag.
- All five have message_count > 0.
- No Nova Carter topic names are present.
- Episode duration is at least 30 seconds.

Exits 0 if all checks pass. Writes:
`results/gnm_fleetsafe_v2_4/episode_001_validation.json`

If any check fails, the episode is invalid. Do not convert it to GNM format.
Record a new episode and validate again.

### Step 4 — Confirm Gate A6

Gate A6 (in `docs/track_a_track_b_completion_gates.md`) passes when:
- `validate_yahboom_episode.py` exits 0.
- The validation report JSON exists with all topics showing message_count > 0.
- The episode metadata JSON exists.

## ROS 2 Recording Commands (reference)

The collection script runs these internally. They are shown here for reference.

```bash
# Record exactly the five canonical topics for 60 seconds.
ros2 bag record \
  /camera/image_raw \
  /odom \
  /tf \
  /scan \
  /cmd_vel \
  --output datasets/gnm_fleetsafe_rosbags/episode_001/rosbag \
  --max-bag-duration 60

# Inspect the recorded bag.
ros2 bag info datasets/gnm_fleetsafe_rosbags/episode_001/rosbag

# Play back the bag to confirm topics replay.
ros2 bag play datasets/gnm_fleetsafe_rosbags/episode_001/rosbag
```

## Topic Remap Note

If the physical Yahboom hardware driver is used instead of Isaac Sim, the camera
publishes on `/camera/color/image_raw` rather than `/camera/image_raw`. Add this
remap to the launch file:

```python
remappings=[("/camera/color/image_raw", "/camera/image_raw")]
```

Isaac Sim publishes on `/camera/image_raw` directly if the OmniGraph node is
configured with `topic_name = /camera/image_raw`.
