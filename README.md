# GNM-VLNVerse Baseline

**Repository:** [FAVL-AI/gnm-vlnverse-baseline](https://github.com/FAVL-AI/gnm-vlnverse-baseline)  
**Author:** Frank Asante Van Laarhoven, Newcastle University  
**Research context:** Doctoral study — GNM visual-goal navigation on VLNVerse/Kujiale indoor scenes

---

## Research scope

This repository provides a reproducible evaluation pipeline for GNM (General Navigation Model)
style visual-goal navigation on VLNVerse/Kujiale indoor trajectory data.

Two evaluation tracks are studied:

| Track | Goal input | Status |
|-------|-----------|--------|
| **Track A** | Goal image provided directly | Completed — stop-policy study with held-out results |
| **Track B** | Language instruction provided | Diagnostic complete — language-dependence not demonstrated; discriminative evaluation pending |

---

## Track A — Stop-policy study

Track A evaluates how well the GNM navigates when given a visual goal image. The primary
finding is that baseline GNM performance is limited by stopping reliability, not navigation
quality.

### Track A results (15 held-out validation episodes)

| Method | SR | OSR | NE (m) |
|--------|---:|----:|-------:|
| GNM baseline | 20.0% | 46.7% | 6.51 |
| Hand-tuned waypoint gate | 26.7% | 26.7% | 5.34 |
| Logistic stop head (train→val) | 20.0% | 46.7% | 6.51 |
| Temporal neural stop head | **33.3%** | 33.3% | **4.47** |
| Geometry-aware oracle (diagnostic) | 46.7% | 46.7% | 3.79 |

These are reproduced experimental results under the documented local evaluation protocol.
The temporal neural stop head improves deployable SR from 20.0% to 33.3% using only
runtime GNM signals.

**Dataset:** 238 train trajectories, 15 validation trajectories across 4 Kujiale scenes
(kujiale_0092, kujiale_0118, kujiale_0203, kujiale_0271).

## Track B — Language-grounding diagnostic

Track B evaluates whether a language instruction can be used to identify a visual goal
image from the trajectory, which is then given to the GNM as a navigation target.

### Instruction source

253 navigation instructions are drawn from [Eyz/VLNVerse_data](https://huggingface.co/datasets/Eyz/VLNVerse_data)
via the VLNTube instube pipeline (V3A Group, University of Adelaide). Instructions were
generated by `gemini-2.5-flash` from rendered trajectory image sequences. They are
**not** human-authored.

### Dataset property

All 253 VLNTube/vlntube_fleetsafe trajectories end exactly at `goal_pos`
(final distance = 0.000 m for every episode by dataset construction). Any retrieval
method that reliably selects the final trajectory frame will achieve SR@3m = 1.000.

### Language-dependence diagnostic (train split, 238 episodes)

| Condition | SR@3m | 95% CI | MRR | Final-frame rate |
|-----------|-------|--------|-----|-----------------|
| correct instructions | 0.987 | [0.973, 1.000] | 0.994 | 0.651 |
| shuffled instructions | 1.000 | [1.000, 1.000] | 1.000 | 0.685 |
| empty instruction | 0.996 | [0.988, 1.000] | 0.998 | 0.874 |
| constant instruction | 0.996 | [0.988, 1.000] | 0.998 | 0.744 |
| random text | 0.992 | [0.980, 1.000] | 0.996 | 0.740 |
| route prior only (semantic=1) | **1.000** | [1.000, 1.000] | 1.000 | 1.000 |
| CLIP only (no route prior) | 0.382 | [0.321, 0.444] | 0.581 | 0.076 |

**Conclusion: LANGUAGE_DEPENDENCE_NOT_DEMONSTRATED**

The route prior alone achieves SR@3m = 1.000. SR@3m remains near 1.000 for shuffled,
empty, constant, and random-text instructions. The correct instruction performs slightly
worse than shuffled (0.987 vs 1.000) because CLIP similarity introduces noise relative
to the pure endpoint prior. Language grounding cannot be claimed on this dataset.

The current route-prior result is strongly influenced by the dataset property that every
recorded trajectory terminates at its geometric goal. It must not be interpreted as
standalone evidence of language understanding.

**Current claim boundary:** Subgoal retrieval SR@3m on vlntube_fleetsafe is determined
by trajectory-endpoint structure, not instruction content. A dataset where trajectories
do not end at the goal is required before language-grounding evaluation can be conducted.

## Installation

### Prerequisites

- Python ≥ 3.10
- PyTorch ≥ 2.2 (CPU or CUDA)
- VLNVerse/vlntube dataset (linked locally — see below)

### Base installation

```bash
git clone https://github.com/FAVL-AI/gnm-vlnverse-baseline.git
cd gnm-vlnverse-baseline
bash scripts/gnm/bootstrap_demo_env.sh
pip install -e .
```

PyTorch is required for model inference. CPU-only environments may install a CPU
build; CUDA is optional. Install it separately if not already present:

```bash
pip install torch torchvision
```

### Language extras (Track B, CLIP retrieval)

```bash
pip install -e '.[language]'
```

## Dataset linking

The VLNVerse/vlntube dataset is not committed to this repository. Link it from your
local copy of FleetSafe-VisualNav-Benchmark:

```bash
bash scripts/gnm/link_vlntube_data.sh /path/to/vlntube
python3 scripts/gnm/check_demo_ready.py
```

For the development workstation:

```bash
bash scripts/gnm/link_vlntube_data.sh \
    ~/robotics/FleetSafe-VisualNav-Benchmark/datasets/vlntube
```

## Track A reproduction

All Track A scripts require a GNM checkpoint (`--ckpt`). Pre-computed results are
committed in `results/bo_reviewer_packet/`.

```bash
# Stop-policy ablation — produces SR/OSR/NE results table (requires checkpoint)
python3 scripts/gnm/ablate_deployable_stop_policy.py \
    --ckpt /path/to/gnm.pth --split val

# Train the logistic stop head (requires checkpoint)
python3 scripts/gnm/learn_stop_head.py --ckpt /path/to/gnm.pth

# Export live dashboard (no checkpoint required)
python3 scripts/gnm/replay_gnm_demo.py --export-live-dashboard

# One-command reproducibility check (no checkpoint required)
bash scripts/gnm/run_reproducibility_pack.sh
```

Evidence: `results/bo_reviewer_packet/`

## Known limitations

- Track B language dependence is not demonstrated on the current dataset. The route
  prior alone achieves SR@3m = 1.000 because trajectories end at goal.
- Track A evaluation is conducted under a local evaluation protocol, not a live
  closed-loop robot deployment.
- Instructions are Gemini-generated (not human-authored); results scope to
  "generated-language grounding", not "human-language generalisation".
- Licence status for upstream datasets (Eyz/VLNVerse_data, Eyz/VLNVerse_scene,
  IAmGoodNavigator, VisualNav-Transformer) is under review. See
  `docs/legal/LICENSING_STATUS.md`.
